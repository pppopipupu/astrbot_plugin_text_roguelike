## 1. 专有代码规范

在编写对决模式相关代码时，必须严格遵守以下模式专有规范：

### 1.1 视觉与渲染规范
- 对决模式公开战局脱敏与各自私密详情渲染。

### 1.2 核心数据与状态规范
- **血量与无职业属性**：对决模式中，双方玩家初始生命值上限与当前生命值均统一为 200 点。无任何子职业系统。
- **TCG 卡牌流转与能量机制**：
  - 玩家回合开始时仅抽 1 张卡牌，且每回合结束时**不丢弃手牌**。
  - 能量成长：第一回合上限为 1A 1BA，后续每 2 个自己的回合增长 1A 能量上限，每 3 个自己的回合增长 1BA 能量上限，最大上限为 10 点，每回合开始时能量重置为当前上限。
  - 后手方在第一回合开始前自动获得 2 个“幸运币”（非卡牌形态），随时可在己方回合打出以增加 1A 动作点。
  - **随从与护符特殊消耗机制**：随从与护符卡牌被打出时，判定为特殊的消耗状态并触发被消耗事件（`CardExhaustEvent`），卡牌 ID 不会进入消耗堆（`exhaust_pile`），也不会进入弃牌堆（`discard_pile`），而是直接从牌组大循环中彻底离场。普通的消耗性卡牌被打出时，依然会正常移入消耗堆并触发消耗事件。
- **召唤失调与攻击重构**：
  - 普通随从召唤进场的回合无法进行攻击（通过将其攻击动作点 `attack_actions` 初始化扣减为 0 来限制，无 `summon_sickness` 实体 Buff）。
  - **突进 (Rush)**：具有突进词条的随从进场当前回合可发起攻击，但目标只能指定为敌方随从（`e2`~`e7`），无法攻击敌方领主（`e1`）。
  - **冲锋 (Charge)**：具有冲锋词条的随从进场当前回合可自由选择攻击领主（`e1`）或随从。
  - **召唤行动点重置**：在下一回合开始（即随从经历回合交替）后，在场我方随从的突进限制解除，攻击动作点重置为 1，可自由选择攻击敌方随从或领主。
  - 随从攻击反击：我方随从攻击敌方随从时，若目标随从在受到伤害后存活，它会对攻击我方的随从发起等同于其攻击力的攻击力反击。
- **伤害卡牌打脸限制**：对决模式中绝大部分法术及攻击卡限制为“只能选择随从为目标”，只有配置声明 `face_target=True` 的直伤卡牌才被允许指定敌方领主为目标。
- **进化点机制**：
  - 双方开局各自拥有 4 个进化点。自第 3 回合开始解锁进化点使用。
  - 每回合仅限进化一次。支持进化手牌、我方随从或我方护符为 `+` 强化版。随从进化时生命补满且上限+2，攻击+2；护符进化时吟唱时间（countdown）保持不减。
- **局外牌组与校验规则**：
  - 开始对决前必须拥有当前活动的合规牌组。
  - 牌组卡牌总数必须在 25~50 张之间，且同名基础卡牌的携带量上限为 4 张。
- **双端渲染与多端显示适配**：
  - 局内发起对战时，群聊中只显示公开的战局简报（血量、能量、随从及护符状态等）。手牌、幸运币和详细指引通过 QQ 私信（DM）下发给双方。
  - 在 Matrix 等不解析纯换行符的第三方聊天平台中，如果消息体是含有 `━━━━━━━━━━━━━━━━━━━━` 分界线的对决公开战局或私密详情，系统会自动检测并整体包裹在 ``` Markdown 代码块中，以确保正常换行和等宽字体排版。

## 2. 测试规范

本项目的对决模式验证应当严格按照自动化测试和边界测试进行：

### 2.1 本地测试手段
- 本地对决模式单元测试：在插件根目录下执行：
  `py -m unittest scratch/test_duel.py`
  验证对决模式的卡组校验、能量成长、幸运币、进化点和对局流程。
