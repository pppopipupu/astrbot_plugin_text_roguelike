# 魔法肉鸽卡牌游戏 机制与指令教程

欢迎来到魔法肉鸽卡牌游戏的世界。本游戏包含两个核心游戏模式：
- 肉鸽模式：经典的文字 Roguelike 卡牌冒险。在随机关卡中进行荒野探索、遗物收集、卡组进化，迎击强大怪物并战胜最终 BOSS！
- 对决模式：双人联机策略卡牌 (TCG) 对决。支持玩家之间构筑牌组、召唤随从、规划能量与动作消耗，并通过进化机制决出胜负。

本教程将为您详细讲解游戏的所有基础指令以及底层的战斗流转、卡牌词条与各种属性机制。

## 1. 指令大全与详细用法

本游戏的所有指令支持中英文变体以及缩写，以下是所有可用指令的详细格式、参数和功能说明。

### 快捷输入与指令简写指南

为了提高游玩的连贯性，游戏支持以下快捷输入与简写机制：
- **首字母简写**：大部分核心指令支持其英文首字母，如打出牌 `p`（play）、随从指令 `m`（move）、选择 `c`（choose）、结束回合 `e`（end）、信息查询 `i`（info）、状态界面 `s`（status）、背包 `inv` / `inventory`。
- **免指令选择**：在事件分支抉择、篝火选项、卡牌发掘选择或主城对话选择等交互提示下，无需输入完整的 `/rogue c 1`，直接在输入框中发送数字（例如：`1`）即可被系统识别为对对应序号选项的确认。
- **免前缀模式**：输入 `/rogue mode` 可切换免前缀模式。启用后，系统会自动检测并拦截以游戏指令、数字或简写开头的消息，使其直接生效，无需输入前缀 `/rogue`。


### start
- 格式：`/rogue start`
- 描述：开启一局新的地下城冒险。
- 特殊场景：若当前已有一局未完成的游戏，直接输入会触发存档覆盖警告。需输入 `/rogue start confirm` 来强制覆盖当前存档并重新开始。

### help
- 格式：`/rogue help`
- 描述：在聊天界面中渲染当前游戏支持的所有核心指令和简要指南。

### tutorial
- 格式：`/rogue tutorial`
- 描述：调出此游戏教程，查看完整的机制说明、指令参数及别名。

### s
- 格式：`/rogue s`
- 描述：在非战斗阶段，该命令将渲染当前玩家所在的地图预览、金币、血量和行动面板；在战斗阶段，该命令将渲染当前的战斗快照（包括你的生命值、动作点、护盾、手牌、战场随从格、敌方单位及意图等）。

### deck
- 格式：`/rogue deck`
- 描述：查看当前玩家的牌组。
- 渲染规则：卡组列表展示时会进行去重，并按照卡牌名称字母升序排序，同时以序号方式呈现。

### overview
- 格式：`/rogue overview [card/relic]`
- 描述：全局查看所有可获得卡牌或遗物的静态属性。
- 示例：
  - `/rogue overview` 或 `/rogue overview card`：查看所有的卡牌和对应的机制属性。
  - `/rogue overview relic`：查看游戏中所有的普通、稀有、珍奇和诅咒遗物的属性与描述。

### stat
- 格式：`/rogue stat`
- 描述：查看玩家生涯的累计统计数据，包括累计输出伤害、累计击败怪物、累计游玩层数以及免前缀模式的状态。该数据属于玩家全局，不会随局内游戏的死亡或放弃而重置。

### class
- 格式：`/rogue class`
- 描述：查看和选择已解锁的子职业。
- 参数：
  - `/rogue class`：列出当前已解锁的子职业列表和当前装备状态。
  - `/rogue class 1`（支持 `c` 代替 `选择`）：装备已解锁的第一项子职业。
  - `/rogue class 2`：装备已解锁的第二项子职业。
  - `/rogue class 0`：取消装备子职业，以普通基础法师模式进行游戏。

### abandon
- 格式：`/rogue abandon`
- 描述：放弃当前的局内游戏。
- 确认机制：输入后系统会发出警告。必须在后面追加 `confirm`，即 `/rogue abandon confirm` 才能彻底删除存档并清空进度。

### mode
- 格式：`/rogue mode`
- 描述：切换当前玩家的“免前缀快捷指令模式”。
- 机制说明：免前缀模式开启后，若你发送的消息以有效游戏指令开头，或者首个词为数字且你拥有活跃存档，该消息将被自动拦截并作为游戏指令执行，无需再加上 `/rogue` 前缀。

### p
- 格式：`/rogue p <手牌序号> [目标]`
- 描述：在战斗中打出指定手牌。手牌序号为 1 至当前手牌上限的数字。
- 目标参数：目标为可选参数。如果是单体效果卡牌且有多个潜在目标，而你没有输入目标，系统会进入等待目标状态，你可以接着输入目标序号（如 `e1` 表示敌人1，`p1` 表示自己或随从1）或发送“取消”/“q”取消本次出牌。
- 随从与攻击型卡牌：当发起攻击但未指定目标时，系统会自动指向第一个存活的敌人。

### m
- 格式：`/rogue m <格子序号|all|*> <动作> <目标/参数>`
- 描述：指挥我方战场上的随从行动。
- 随从格子：格子序号为 1 到 6 的数字。可以使用 `all` 或 `*` 批量指挥所有存活随从。
- 动作：支持 `a` (攻击) 和 `s` (技能)。
- 示例：
  - `/rogue m 1 a e1`：指挥 1 号位的随从攻击 1 号位的敌方目标。
  - `/rogue m all a`：批量指挥所有存活随从攻击敌方默认目标。

### c
- 格式：`/rogue c <序号>`
- 描述：在遭遇关卡契约、事件选项或选择卡牌时，确认你的选择。在符合条件的场景下，你也可以直接发送数字序号。

### sa
- 格式：`/rogue sa <手牌序号> [目标]`
- 描述：打出或释放对应手牌的特殊行动。

### e
- 格式：`/rogue e`
- 描述：结束当前你的战斗回合，将回合交由敌方行动。

### f
- 格式：`/rogue f` (别名：`折叠`, `fold`)
- 描述：切换是否在每一次渲染的底部折叠显示操作指南列表，方便熟悉游戏后的简洁显示。

### q
- 格式：`/rogue q <[指令1, 指令2...]>` (别名：`队列`, `queue`)
- 描述：将多条战斗指令通过逗号分隔打包在方括号中，一次性顺序执行。即便其中某条指令执行失败，也会强行继续执行后续指令。
- 示例：`/rogue q [p 1, m 1 a e1, e]`

### qi
- 格式：`/rogue qi <[指令1, 指令2...]>` (别名：`中断队列`, `queue_interrupt`)
- 描述：将多条指令通过逗号分隔打包在方括号中顺序执行。与 `q` 的区别在于，如果在执行过程中任何一条子指令执行失败，系统会立即中断后续所有剩余指令的执行。
- 示例：`/rogue qi [p 1, e]`

### i
- 格式：`/rogue i [参数]` (别名：`查询`, `query`, `info`)
- 描述：获取战斗信息或模糊查询数据。
- 参数细分：
  - 不带参数时：在战斗中输入可渲染详细战场状态。
  - 牌堆查询：`/rogue i draw`、`/rogue i discard`、`/rogue i exhaust`。
  - 墓地查询：`/rogue i mg`、`/rogue i eg`。
  - 模糊匹配描述查询：`/rogue i <名字>` 能够模糊查询卡牌、随从、遗物、护符、Buff 或怪物的机制解释与具体描述，例如 `/rogue i 眩晕`。

### bag
- 格式：`/rogue bag` (别名：`背包`, `inventory`, `inv`)
- 描述：查看当前拥有的主城背包物品及金币 (GP) 资产。

### quest
- 格式：`/rogue quest` (别名：`任务`, `quests`)
- 描述：查看当前已承接的主城任务列表、任务达成进度以及奖励信息。

### lock
- 格式：
  - `/rogue lock`：查看当前卡牌锁定管理系统状态和已锁定列表。
  - `/rogue lock <卡牌名称>`：锁定指定卡牌以便在开局时必带。
  - `/rogue unlock <卡牌名称>`：解除对指定卡牌的开局锁定。
- 描述：在主城模式下，用于管理开局时必带的卡牌，最多可锁定 8 张卡牌。

### map
- 格式：`/rogue map` 或 `/rogue map [classic/radar]` (别名：`地图`)
- 描述：在主城模式下，切换主城渲染的地图样式，支持“经典全局大地图”（`classic`）与“局部雷达十字小地图”（`radar`）。

### draw
- 格式：`/rogue draw` (别名：`抽牌堆`, `draw_pile`)
- 描述：在战斗中，直接查询并列出当前抽牌堆中剩余的卡牌。

### discard
- 格式：`/rogue discard` (别名：`弃牌堆`, `discard_pile`)
- 描述：在战斗中，直接查询并列出当前弃牌堆中的卡牌。

### exhaust
- 格式：`/rogue exhaust` (别名：`消耗堆`, `exhaust_pile`)
- 描述：在战斗中，直接查询并列出当前已被消耗的卡牌堆。

### mg
- 格式：`/rogue mg` (别名：`随从墓地`, `minion_graveyard`)
- 描述：在战斗中，直接查询并列出我方当前已死亡的随从与护符。

### eg
- 格式：`/rogue eg` (别名：`敌人墓地`, `enemy_graveyard`)
- 描述：在战斗中，直接查询并列出已击杀的敌人墓地。

### talk
- 格式：`/rogue talk <目标名称>` (别名：`交互`, `talk_to`, `interact`, `inter`)
- 描述：与主城或咖啡馆中的 NPC / 顾客进行交互对话。例如 `/rogue talk 铁匠`。

### take
- 格式：`/rogue take <物品名称>` (别名：`拿取`, `pick`)
- 描述：拿取主城当前房间场景中的地面物品。例如 `/rogue take 幸运硬币`。

### use
- 格式：`/rogue use <物品名称>` (别名：`使用`)
- 描述：使用主城背包中的特定物品。例如 `/rogue use 遗失的笔记本`。

### look
- 格式：`/rogue look` (别名：`看`, `查`, `look_around`)
- 描述：在咖啡馆中环顾四周，查看当前咖啡馆的环境状况与在店顾客。

### leave
- 格式：`/rogue leave` (别名：`离开`, `退出`, `返回`)
- 描述：在咖啡馆中结束与当前 NPC 正在进行的对话交谈。

### exit / quit / 取消
- 格式：直接发送 `exit`、`quit` 或 `取消`
- 描述：在临时拦截状态下（如等待单体卡牌选择目标、等待卡牌发掘选择、以及霸瞳天星的卡牌选择等），打出该命令会中断并退出当前的动作，无损地退回到正常的战斗决策阶段。

## 2. 基础游戏运作机制

本游戏包含卡牌牌组运转、动作资源管理以及关卡推进三个基础维度。

### 牌组与战斗循环
- **回合抽牌**：每回合开始时，玩家默认从抽牌堆中抽取 **6 张** 新卡牌。特别地，在战斗的首个回合开始前，系统会先将玩家的整套牌组洗入抽牌堆中再抽取起手牌。
- **手牌上限**：玩家默认手牌上限为 **12 张**（某些特定效果可能使上限降低）。若抽牌时手牌已达到上限，多余抽取的卡牌将被直接浪费移入弃牌堆。
- **弃牌堆与重洗**：使用过的卡牌、未保留的卡牌均会进入弃牌堆。当玩家需要抽牌但抽牌堆已空时，系统会自动将弃牌堆中的卡牌重新洗牌并放入抽牌堆中，实现无限循环。
- **回合结束弃牌**：玩家回合结束时，手牌中不具备保留（retain）效果的所有卡牌将被自动丢弃并移入弃牌堆。特别注意：回合结束自动丢弃的操作绝不会触发卡牌的灵巧（agile）效果。
- **消耗堆**：使用后具有“消耗”特性的卡牌使用后会进入消耗堆，在当前战斗中无法被重新洗回。

### 动作点资源 (A/BA)
- 玩家每回合开始时重置动作资源。基础积蓄为 2 个动作（A）和 1 个附赠动作（BA）。
- 打出卡牌会扣除卡牌对应的 A 或 BA 消耗。
- 免费卡牌：若卡牌属性中动作消耗为 0A 0BA，打出时不消耗任何动作资源。
- X 费卡牌：若卡牌的动作消耗显示为 `X A`、`X BA` 或 `X A Y BA`，打出时会扣除并消耗当前玩家所拥有的全部 A 或 BA（即扣减为 0）。卡牌产生的效果会根据本次实际扣减的动作点数量进行成倍放大（如果您拥有特定遗物如【X化学物】，结算时的计数还会额外 +2）。

### 生命与护盾
- 生命值（HP）：代表玩家或敌人的生命上限。生命归零时冒险失败。
- 护盾（Shield）：在受到非真实伤害时，优先由护盾扣减。
- 护盾流失：在各自回合开始时，玩家和所有存活敌人的护盾均会自动失去一半（数值向下取整）。
- 胜利判定：当存活的敌人全部被击败，或者当存活敌人全部被判定为爪牙（召唤物）时，战斗自动宣告胜利。

## 3. 卡牌词条与机制详解

卡牌具备不同的卡牌类型，并可触发各种被动/主动词条效果。

### 卡牌类型与行为
- **法术 (spell)**：直接打出并执行法术伤害、给予护盾或施加各种状态，结束后移入弃牌堆。
- **能力 (ability)**：使用后赋予玩家永久的被动能力（以 Buff 呈现），当前战斗一直生效。
- **随从 (minion)**：
  - 随从被打出召唤时进入战场，并触发独有的“入场曲”。
  - 战场最多容纳 6 个随从和护符（格子编号 1 到 6，对应指向为 p1 到 p6）。
  - 随从召唤被打出时，被判定为特殊的消耗状态并触发 CardExhaustEvent，但不进入消耗堆，也不进入弃牌堆，从牌组循环中彻底离场。
  - 随从拥有独立行动点。回合交替时，存活随从的动作点（actions）累加 1。随从的攻击动作点（AA）每回合重置为 1。只有特定奥术随从的额外动作点（bonus_actions）会累加 1。非珍奇级别随从无附赠动作。
- **护符 (amulet)**：
  - 护符打出后进入战场，拥有初始的“吟唱时间”。
  - 回合更替时，护符的吟唱时间会流失（每回合减少 1）。
  - 当吟唱时间归零销毁时，该卡牌会自动被移入弃牌堆（进入牌组循环），并触发独特的“谢幕曲”（Last Words）效果（如获得金币、护盾、治疗或真实伤害等反馈）。

### 核心卡牌词条
- **保留 (retain)**：拥有该效果的卡牌在回合结束时不会被自动丢弃，而是保留在手牌中。
- **灵巧 (agile)**：在卡牌因其他卡牌或效果被主动“丢弃”时触发特殊效果。回合结束时的系统自动丢弃不触发此词条。
- **消耗 (exhaust)**：卡牌打出后从本场战斗的牌组循环中离场，进入消耗堆。
- **真实伤害 (true)**：特殊伤害类型。造成的伤害无视并完全穿透护盾，直接扣减生命值。
- **回响形态 (echo_form)**：在本回合中，你打出的前几张牌会获得回响（额外自动打出）。每张牌最多回响 10 次，多余的层数会顺延至后续打出的卡牌。
- **战斗专注 (tactical_focus)**：打出后获得大量抽牌，但会给自己附加“无法抽牌”状态（在状态生效期间，你无法通过任何卡牌或随从效果进行抽牌，每回合结束时层数减 1）。
- **X 费卡牌 (X-cost)**：卡牌的动作消耗被设为 `-1`。打出时会自动扣除当前拥有的全部对应动作资源作为 X（或 Y）计数，并根据该计数成倍放大卡牌效果（如多次真实伤害、多倍护盾、多次随从召回等）。如果拥有遗物【X化学物】，结算计数还会额外 +2。

### 状态效果词条
- **力量 (strength)**：提高造成的物理或法术伤害数值。
- **易伤 (vulnerable)**：角色受到的某种属性伤害翻倍（轻度易伤增加 50%）。
- **抗性 (resist)**：角色受到的某种属性伤害减少 50%。
- **免疫 (immune)**：角色受到的某种属性伤害减少 100%（不受该属性伤害）。
- **眩晕 (stun)**：处于该状态下的敌人在下一回合无法行动，并在回合更替时消耗一层眩晕层数以恢复。
- **电击 (shock)**：受到的闪电和雷鸣伤害每层增加 1 点。
- **闪电护体 (lightning_shield)**：受到伤害时向攻击者反弹闪电伤害并施加电击状态。
- **燃烧 (burning)**：受到的灼烧折磨，在每回合开始时会受到火焰属性伤害。
